# 模式 B — 用户离场自动循环

详见 [protocol_redline](./protocol_redline.md) "模式 B 沙盒" 部分.

本文档展开模式 B 的完整流程 + 边界 + 例外.

## 触发进入模式 B

任一触发即进入:

1. 用户明说 "自动跑" / "无人值守" / "auto" / 类似
2. 用户连续 5 次 outbox 心跳 (~5 min 每次) 无回应 + 当前研究 batch 完成 + 模式 B 上限未达

(per [feedback-auto-shutdown-5loops](../../.claude/memory/feedback_auto_shutdown_5loops.md), 5 次心跳是已有的"用户离场"判定标准)

## 30 分钟无回复默认动作

这不是完整模式 B.

当系统明确询问用户下一步, 且 30 分钟内没有回复:

- AI 先读 3 个入口文件, 填写 `当前策略`.
- 默认保持 `当前策略`.
- 默认换一个研究方向.
- 不默认归档当前策略.
- 不默认升级实盘.
- 不继续在刚失败的同一方向上调参数.
- 新方向必须来自 `当前策略` 的当前状态、失败原因和未完成线索.
- 先写新方向设计; 若需要新增数据源、改变核心代码、提高预算, 停下等用户.

## 进入模式 B 前的前置检查 (硬约束)

进入前 AI 必须确认:

- [ ] 经验账本"未完成线索"或"未来探索方向"至少有 1 条优先级 ≥ 中
- [ ] 累计算力花费 < ¥20 (模式 B 总预算上限)
- [ ] 累计模式 B 跑过的轮数 < 5
- [ ] 用户离场时长 < 12 小时
- [ ] 协议红线版本和 Codex 同步

任一不满足 → **不进入模式 B**, 写 autonomous_summary 等用户回来.

## 模式 B 单轮流程

每一轮 AI 自驱走完 6 步, 严格遵守协议红线 + 沙盒约束:

1. **挑下一个**: 从经验账本按优先级挑一个研究方向
2. **设计实验**: 按 spec_template 写实验设计文档,包括 8 必填字段
3. **Codex 检查 + 跑**: Codex 收 DIRECT 后跑预检 schema, 通过后开跑
4. **数据出**: Codex 出 4 类 artifact (ranked + trades + daily_equity + trigger_dates)
5. **跑标准疑点 + 论证**: Claude 跑 Q1-Q5 (有新机制时 Q6-Q7), Codex 回应
6. **写报告**: Claude 写复盘报告,三个出口选一:
   - 采用 → 写入经验账本"已采用方向"
   - 拒绝归档 → 写入经验账本"已确认无效"
   - 再做一轮 → 写迷你 spec 回第 1 步 (但同样受所有上限制约)

## 沙盒约束 (再次明确)

模式 B 中, AI **绝对不能**:

- B1 改代码 — 任何 `.py` 文件
- B2 加新数据来源 — 任何不在 yaml 绿区已声明的 parquet/CSV
- B3 加新参数到 yaml 黄区 — 必须用户在场才能 PR
- B4 改协议红线本身 — 必须用户在场拍板
- B5 碰经验账本"已确认无效"区域 — 永远禁区

如果 AI 在模式 B 中**发现需要做** B1-B5 之一才能继续 (e.g. 当前研究必须改代码), 立刻写 `MODE-B-ESCALATE` 消息, 停下来等用户回来.

## 停止条件 (任一触发)

模式 B 持续到下面任一触发, 然后 **强制停, 写汇总, 关 spot, 等用户**:

| 上限 | 阈值 | 来源 |
| --- | --- | --- |
| 累计算力花费 | ¥20 | 用户设定 (2026-05-14) |
| 累计轮数 | 5 | 用户设定 |
| 用户离场时长 | 12 小时 | 用户设定 |
| 可探索列表 | 已空 | 经验账本状态 |
| 任何 B 沙盒越界 | 立即 | 协议红线 |
| 用户回来发任一消息 | 立即 | 切回模式 A |

## 停止后必做

- 关 spot VM (如果还开着): `ssh -i ~/.ssh/quant_spot.pem ubuntu@<spot-ip> 'sudo shutdown -h now'`
- 写 [autonomous_summary](./autonomous_summary_template.md) 报告
- 经验账本所有变更 commit 到 git
- 等用户回来审查

## 用户回来后

用户看 autonomous_summary 报告:

- 期间跑了 N 轮 (列出每轮研究问题 + 结论)
- 总花费 ¥X / 总用时 Y 小时
- 经验账本变更 (新增已采用/已确认无效/未完成线索)
- 剩余预算 ¥(20-X)

用户可以:
- ✅ 全部接受 → 继续往下
- 🔄 推翻某轮结论 → AI 把该轮重新归档
- ➕ 补充经验账本
- 🔧 调整模式 B 阈值 (¥20 / 5 轮 / 12 小时)
- ⏸️ 关闭模式 B → 后续 5 次心跳无回应不再触发, 必须用户明说才进入

## 异常处理

- **算力账户欠费 / spot 起不来**: 当前轮归档为 "中断", 写汇总等用户
- **VM 网络断**: 等 3 次重试, 仍不通则归档当前轮, 写汇总
- **协议红线版本不一致**: 暂停模式 B, 等用户回来同步版本
- **经验账本格式损坏**: 暂停, 写 ESCALATE 消息

## 启动 / 停止命令 (用户控制)

用户在场时, 任一指令切换:

- 启动: "自动跑" / "无人值守" / "开启模式 B"
- 暂停: "暂停" / "停" / "关闭模式 B"
- 调整阈值: "把模式 B 预算改成 ¥XX" / "把模式 B 轮数改成 N"

用户不在场时, 模式 B 按上述规则自驱.
