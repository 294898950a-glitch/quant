# 模式 B 结束汇总报告模板

模式 B 触发任一停止条件后, Claude 写一份此报告, 等用户回来审.

存放位置: `reports/autonomous_<YYYY-MM-DD>_<batch_id>.md`

## 模板

```markdown
# 模式 B 自动循环汇总

日期: YYYY-MM-DD 离场 → YYYY-MM-DD 回来
持续时长: HH:MM
触发原因: <下面任一>
  - 累计算力 ¥X >= ¥20
  - 累计 N 轮 >= 5
  - 用户离场 X 小时 >= 12 小时
  - 经验账本"可探索"列表空
  - 沙盒越界 (具体哪条)
  - 用户回来主动触发

## 期间研究汇总

总共跑了 N 轮:

| 第 # 轮 | 假设 | 用时 | 算力 | outcome | 审计 verdict |
| --- | --- | ---: | ---: | --- | --- |
| 1 | medium signal recovery_days 调整 | 75 min | ¥5 | adopted | healthy |
| 2 | switch_hurdle_pct 网格 | 90 min | ¥6 | rejected | stagnant |
| ... | ... | ... | ... | ... | ... |

总算力: ¥X (预算上限 ¥20, 剩余 ¥(20-X))

## 每轮详情

### 第 1 轮: <一句话假设>

- **结论**: adopted / rejected / reroute
- **关键数字**: replay_2020 = X, replay_2021 = Y, ...
- **审计 verdict**: healthy / stagnant / data_mining / diverging
- **报告**: reports/<...>.md
- **5 项质疑通过情况**: ✓✓✓⚠✓ (具体哪项 warning + 原因)

### 第 2 轮: ...

(每轮独立小节)

## 经验账本变更

期间累计:
- 新增"已采用方向" N 条 (列出)
- 新增"已确认无效方向" M 条 (列出)
- 新增"未完成线索" K 条
- "未来探索方向" 划掉 L 条 (已尝试过)

## 审计员汇总

整体 trajectory 判定:
- N 轮全 healthy → 模式 B 运行良好
- 中途出现 stagnant/data_mining/diverging → 列出第几轮, verdict 详情
- 否决导致退出 → 哪轮否决, 否决原因

## 沙盒边界遵守情况

模式 B 沙盒约束 B1-B5 是否被触碰:
- B1 (改代码): ✓ 未触碰
- B2 (加新数据): ✓ 未触碰
- B3 (新参数到黄区): ✓ 未触碰
- B4 (改协议红线): ✓ 未触碰
- B5 (碰已确认无效): ✓ 未触碰

如有触碰 → ESCALATE 详情 + 退出原因.

## 算力账单

| 项 | 用时 | 花费 |
| --- | ---: | ---: |
| spot VM | X 小时 | ¥Y |
| sig VM (长期开, 边际 0) | - | - |
| **总计** | **X 小时** | **¥Y** |

预算: ¥20
已用: ¥Y
剩余: ¥(20-Y)

## 异常和升级

期间有没有发生:
- 算力账户欠费
- VM 网络断
- 协议红线版本不一致
- 经验账本格式损坏
- 任何 MODE-B-ESCALATE

如有 → 详情 + 处理方式.

## 给用户的建议

(Claude 主观判断, 用户可全部接受 / 推翻 / 调整)

- 是否调整模式 B 阈值 (¥20 / 5 轮 / 12 小时)?
- 经验账本里哪些"未完成线索"建议下次优先?
- 哪些"已确认无效"建议复查?
- 有没有发现新的"未来值得探索方向"?

## 用户审查后行动

用户可以:
- [ ] 全部接受 → 经验账本已合并, 继续往下
- [ ] 推翻第 N 轮结论 → 重新归档
- [ ] 调整模式 B 阈值
- [ ] 关闭模式 B
- [ ] 补充经验账本

## 下一步建议

(模式 B 退出后, 下一个研究方向)
```

## 必填字段

跟 L6 复盘报告类似, autonomous_summary 也有必填段:

- ❌ 不能省"期间研究汇总" — 否则用户不知道发生了啥
- ❌ 不能省"经验账本变更" — 否则记忆系统失同步
- ❌ 不能省"沙盒边界遵守情况" — 否则用户没法验证 AI 守了协议
- ❌ 不能省"算力账单" — 否则预算追不上
- ❌ 不能省"审计员汇总" — 否则用户没法判断 trajectory 是否健康
- ✅ 弹性: 异常段、给用户的建议段、研究细节深度

## 写入触发

任一停止条件触发后, Claude 必须**立即写**:
1. 关 spot VM (per [feedback_auto_shutdown_5loops](../../.claude/memory/feedback_auto_shutdown_5loops.md))
2. 写 autonomous_summary 到 reports/
3. commit 到 git
4. 在 outbox 写"模式 B 已退出, 等待用户回来" 消息
