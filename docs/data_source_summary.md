4. 强赎公告/强赎状态 (redeem_status) 历史时间线
5. 覆盖范围: 2022-2026年

四、AKShare (开源免费)
----------------------
关键接口评估：

1. bond_zh_cov_value_analysis(symbol="113527")
   - 日频数据：日期, 收盘价, 纯债价值, 转股价值, 纯债溢价率, 转股溢价率
   - 覆盖：上市日~退市/到期（仍交易中的债券可到2026年）
   - 优点：含转股溢价率时序数据
   - 缺点：每次只能查一只债券，无剩余规模，无强赎状态
   - 速度：慢（逐个HTTP请求）

2. bond_zh_hs_cov_daily(symbol="sh113504")
   - 日频数据：date, open, high, low, close, volume
   - 覆盖：上市日~退市
   - 优点：长时间覆盖
   - 缺点：无溢价率，无剩余规模，无强赎状态

3. bond_cb_redeem_jsl() - 集思录强赎/回售快照
   - 实时快照：341只债券, 含"强赎状态"列
   - 状态值：已公告强赎, 公告不强赎, 公告要强赎
   - 含：剩余规模, 强赎天计数, 强赎触发价, 强赎条款
   - 缺点：实时快照，非历史时序，不可用于回测

4. bond_cb_jsl() - 集思录可转债实时列表
   - 实时快照：30只（有限），含转股溢价率、剩余规模

5. bond_zh_cov_info() - 东方财富转债基本信息
   - 含：IS_REDEEM(是否触发强赎), REDEEM_TRIG_PRICE, IS_SELLBACK
   - 静态信息，非历史

结论：AKShare 全部是实时/当前数据，无历史强赎状态时序回放能力。

五、国内量化平台
----------------
1. JoinQuant (聚宽) - www.joinquant.com
   - 提供可转债日线数据（get_price支持可转债代码）
   - 有转股溢价率、纯债价值等因子
   - 强赎状态需自行从公告数据推算
   - 限制：需在聚宽平台内回测，本地使用需下载数据

2. RiceQuant (米筐) - www.ricequant.com
   - 类似JoinQuant，支持可转债
   - 数据可下载用于本地回测（有数据导出功能）
   - 有可转债专题数据包

3. Wind (万得) - 收费，企业级
   - 最完整的数据源
   - 可转债历史日线、转股溢价率、剩余规模日频、强赎状态
   - 全部需要的数据都有，但收费昂贵

4. Choice (东方财富) - 收费
   - 类Wind，数据覆盖较全
   - 有可转债专题

六、可转债专业网站
-----------------
1. 宁稳网 (www.ninwin.cn) - 国内最完整的可转债数据网站
   - 提供可转债历史日线下载（CSV格式）
   - 含转股溢价率、纯债价值、到期收益率等
   - 有强赎状态记录
   - 可能有强赎历史数据
   - 需注册，部分功能付费

2. 集思录 (www.jisilu.cn) - 国内主流可转债社区
   - 提供可转债实时数据（通过AKShare可获取）
   - 历史数据可通过爬虫获取，但无公开API
   - 强赎状态实时更新，但历史状态未公开提供

七、核心结论与推荐方案
---------------------
方案A：当前数据仓库已足够（推荐，最快路径）
  已有数据可以直接支撑2022-2026年回测：
  - 日线行情 ✅ (cb_daily: close, cb_over_rate)
  - 转股溢价率 ✅ (cb_over_rate 日频)
  - 强赎公告时间线 ✅ (cb_call: ann_date + is_call 状态)
  - 强赎触发进度 ✅ (正股日线 + 转股价，精确计算)
  - 缺失：每日剩余规模历史 → 可用发行规模 - 累计赎回量估算

  回测思路：
  1. 对每个交易日，用 cb_call 公告历史 + 正股日线计算强赎状态
  2. 在公告"已满足强赎条件"或正股触发条件达标时，标记为"强赎即将发生"
  3. 在"公告实施强赎"日，记录最终强赎登记日和价格
  4. premium_ratio 直接用 cb_over_rate

方案B：补充数据源（增强版）
  1. 通过 bond_zh_cov_value_analysis 补充部分债券的溢价率校验
  2. 通过宁稳网爬取历史剩余规模数据（如有）
  3. 考虑JoinQuant数据导出

方案C：付费数据源
  1. Wind/Choice - 最完整但贵
  2. JoinQuant聚宽 - 可在平台内回测，年费约1000-2000元

八、建议
-------
当前数据仓库已覆盖所有核心需求。主要缺失的每日 remain_size 历史，
可以通过以下方式解决：
1. 使用 cb_basic 的 remain_size 作为最新值，回测中假设规模线性递减
2. 使用 cb_call 中的 call_vol 累加推算剩余规模变化
3. 忽略 remain_size 的动态变化（对于强赎博弈策略，强赎状态和溢价率比剩余规模更关键）

建议优先使用方案A做快速回测验证，后续再补充remain_size数据增强精度。
