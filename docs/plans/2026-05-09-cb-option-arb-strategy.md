# 可转债期权定价套利策略 — 设计文档

**实验 #2,起于 2026-05-09(网格实验封档后第二天)**

---

## 一句话

可转债 = 债 + 一张可以转股的看涨期权。算出**理论该值多少钱**,跟市场价比,**便宜就买,贵了就卖**。等市场把误差纠正过来赚价差。

## 为什么这次大概率比上次靠谱

```
网格(实验 #1)      没有"该值多少钱"的锚点, 只赌价格在区间内来回 → 单边市必死
期权套利(实验 #2)   有 *绝对估值锚点* (理论价) → 不依赖市场震荡, 靠定价偏差赚钱

网格在 2022-2026 中美蓝筹全输持有的根本原因: 市场涨它也涨不动, 还少赚分红.
期权套利的市场基础: 中国可转债市场散户多 + 流动性差 → 定价偏差经常出现, 这是 alpha 来源.
```

---

## 第一部分:可转债是个什么东西(后来者入门)

可转债是公司发的一种**特殊债券**,核心条款 4 条:

```
1. 票面利率                每年付的利息(通常 0.5%~3%, 比一般公司债低)
2. 转股权                  持有人可以按"转股价"把债换成正股, 不行使就到期收面值
3. 强赎条款                公司可强制按面值买回:正股价持续 ≥ 转股价×130% 的 30 个交易日里有 15 天
4. 下修条款                公司有权下调转股价(刺激持有人转股, 减少还本压力)
```

价格特性:

```
低位有"债底": 即使正股跌穿了, 持有到期还能拿回面值 + 利息 → 价格不会跌穿债底
高位有"强赎封顶": 正股大涨触发强赎后, 公司用面值赎回, 价格被锁 → 不会涨过 ~130 元
中段是 "期权" : 正股越接近转股价, 转股的期权价值越高
```

**就是这个"债底有保底 + 强赎有顶 + 中段是期权"的不对称结构,让 CB 有定价偏差的空间**。

## 第二部分:理论价怎么算

### 拆成两块

```
CB 理论价 = 债底现值 + 转股期权价值
```

### 债底现值

把所有未来现金流(每年利息 + 到期本金)按某个折现率打回今天:

```
PV = Σ (利息_t / (1+r)^t) + (面值 / (1+r)^T)

r 用什么:  无风险利率 + 信用利差 (评级 AAA 用 +0.5%, AA 用 +1.5%, A 用 +3%, 之类)
```

### 转股期权价值

最简单先用 **Black-Scholes 欧式看涨期权公式**(v1):

```
输入:
  S        正股当前价
  K        转股价 (会因下修变化, 用最新值)
  T        到期还有多少年
  σ        正股波动率 (用历史 30 / 60 / 120 日实现波动率)
  r        无风险利率
  
输出: 一份正股看涨期权的价值
然后: 期权价值 × 转股比例 = 这只 CB 的"期权部分"价值

其中转股比例 = 100 (面值) / K
```

v2 升级:用**二叉树定价**,能 handle 美式提前行权 + 强赎触发条件 + 下修条款。这次先做 v1,验证可行后再说 v2。

### 注意几个 corner case

```
1. 还在转股期前       期权暂时不能行权, 但仍有时间价值 → 仍按 BS 算, T 不变
2. 临近到期(< 30 天)   期权价值约等于 max(S-K, 0), BS 失效需要直接算 intrinsic
3. 已经强赎触发        理论价直接锁在 ~103 元 (面值 + 利息) — 强赎后没期权价值了
4. 下修发生            K 改了, 当时算的理论价立即更新
5. 回售期              价格不会跌穿 100 元 (持有人可按面值卖回)
```

## 第三部分:买卖信号

### 买入条件(同时满足)

```
1. 市场价 < 理论价 × (1 - 低估门槛)        默认低估门槛 5%, 可调
2. 该 CB 不在强赎触发期内(否则估值锁顶)
3. 距到期 > 30 天(否则 BS 失效)
4. 当前持仓 < 单只仓位上限(默认 3%, 可调)
5. 总持仓数 < 最大持仓数(默认 30, 控制集中度)
```

### 卖出条件(任意满足)

```
1. 市场价 ≥ 理论价 × (1 + 公允门槛)         默认公允门槛 0%, 即一回到公允就卖
2. 触发强赎(估值上限被锁, 没继续上涨空间)
3. 持有超过最大天数(默认 90 天, 防永远不收敛)
4. 距到期 < 30 天(避免到期前 BS 失效带来的不可控)
5. 单只浮亏超过止损线(默认 -8%, 防 CB 评级跳水或正股暴跌)
```

### 仓位规模

```
单笔买入 = 总资金 × 单只仓位上限 (默认 3%)
最大同时持仓 = 30 (覆盖率高, 单只崩对组合影响小)
满仓约 = 30 × 3% = 90% 资金参与, 留 10% 现金应付波动
```

## 第四部分:标的池过滤

不是每只 CB 都该进策略池。每天开盘前过滤:

```
评级 ≥ AA-                   防垃圾债
剩余规模 > 1 亿元             防流动性陷阱
日均成交额 > 100 万元(20 日均)  防进得去出不来
不在强赎触发计数中             已锁顶, 没期权价值
不在最后 30 天                 BS 失效
不在转股期前(可选, 看有没有时间价值)
```

过滤后日均池子大概 300-500 只(全市场约 1200 只 CB,大头都是过得了筛的)。

## 第六部分:不可调(策略骨架,红线)

```
Black-Scholes 公式本身                          数学
债底现金流计算(面值 + 利息按合同打回)            合同规定
"市场低于理论价 × (1-X%) 则买" 这个核心 if-else  策略论点
回测引擎                                        框架级, 不动
holdout 池子设计                                防过拟合机制
```

LLM 不能改这些,只能在 13 个参数空间里搜。

## 第七部分:**先验证定价引擎,再做策略**

跑策略前先做一个 sanity check 实验。这步**很重要**——如果定价引擎本身有 bug,再聪明的策略都救不回来。

### Sanity check #1:对历史强赎事件

```
取过去 5 年所有发生过强赎的 CB(已知 ~200 个)
往前回看 30 天:
  这 30 天内 CB 价格应当从波动 → 收敛到 ~103 元 (面值 + 一点利息)
  我们用 BS 算的理论价应当也跟着收敛到 ~103 (强赎条件触发后 期权价值 → 0)
  
检查:    实际市场价收敛轨迹  vs  我们的理论价收敛轨迹  应该高度重合
若不重合: BS 没考虑强赎条款 → 对啊, BS 是欧式 → 这是预期内的, 标记 corner case
```

### Sanity check #2:静态点估

```
取一只典型 CB: 比如 2023 年中期某天的 招商转债
查这天的:
  正股价
  转股价
  剩余天数
  HV60 波动率
  无风险利率(国债 5 年)
  评级 AAA → 信用利差 50bp
  
用 BS + 债底, 算出理论价
跟当天实际收盘价比  
差距 < 5% → 引擎大致对得上
差距 5-15% → 可解释(市场情绪偏离)
差距 > 15% → 引擎有 bug 或错配
```

### Sanity check #3:横截面散点

```
取某一天全市场 ~300 只活跃 CB
对每只算理论价 vs 市场价 的散点图
理想: 沿 45 度线分布(理论 = 市场), 误差±10% 内
异常: 散点远离 45 度线, 散得到处都是 → 引擎有问题
```

**只有这 3 个 sanity check 通过,才进入策略层(信号 + 回测)。**否则定价错的话回测出来的 sharpe 都是骗人的。

```

```

不再跟"个股持有"比 — CB 跟个股不可比。比同类指数才公平。

## 第九部分:预期失败模式 + 怎么早期发现

```
失败模式 A:  BS 公式低估了下修条款的价值
            → 表现: 大量 CB 被算成"低估", 全买入后市场没回调, 持续浮亏
            → 早期信号: sanity check #3 显示"市场普遍比理论价高 5-10%"
            → 修法: 上 v2 二叉树, 显式 handle 下修

失败模式 B:  历史波动率低估了未来波动率
            → 表现: 期权价值算低, 理论价偏低 → 大量"看似低估"实际公允
            → 早期信号: 跨年看, 跟随波动率走高的窗口里, 假信号集中
            → 修法: vol_multiplier 调高, 或换隐含波动率(IV)估计

失败模式 C:  评级跳水的 CB(债底骤降)
            → 表现: 个别 CB 实际崩盘 -30%, 拉低组合
            → 早期信号: 单笔止损线被触发的频率
            → 修法: rating_floor 调高(只玩 AAA / AA+) 或动态评级监控

失败模式 D:  流动性陷阱
            → 表现: 想卖卖不掉, 只能跌停板挂单, 实际成交价远低于 close
            → 早期信号: 回测里的 fill 假设 vs 真实成交差距大
            → 修法: 加大日均成交额过滤阈值, 或建模引入冲击成本

失败模式 E:  跨市场风险传染(2022 那种全面熊市)
            → 表现: CB 跟随股市暴跌 + 评级集体跳水 → 双杀
            → 早期信号: 跟中证转债指数对比, 大幅下行段显著跑输
            → 修法: 加 macro filter (大盘极端弱时减仓 / 暂停)
```

每一种失败都有可观察的预警信号。回测中要专门看这些指标,而不是盯单一 sharpe。

## 第十部分:成功的定义

不要只看一个夏普。要满足全部条件:

```
1. 跨 8 个 OOS 池子的 sharpe 一致性: std(sharpe) < 0.5
2. 4 年总收益 跑赢 中证转债指数 ≥ 5%
3. 最大回撤 < 中证转债指数 同期最大回撤
4. 审计员不判过拟合
5. sanity check #1/#2/#3 全部通过
6. 失败模式 A-E 没有显著触发
```

任何一条不满足 → 还不能上市场。

## 第十一部分:第一步具体动作

```
Step 1: 重拉 cb_warehouse 数据 (~2 小时, build_cb_warehouse.py 跑一次)
Step 2: 写期权定价引擎 (cb_pricer.py): BS + 债底 + 5 个 corner case 处理
Step 3: 跑 sanity check #1/#2/#3 (一个独立脚本, 不进策略)
Step 4: ↓ 引擎正确才往下
        ↓
Step 5: 写策略 verifier (cb_option_arb/verifier.py): 信号 + 投资组合
Step 6: tunable_space.yaml + bootstrap + orchestrator_main
Step 7: 部署 vm + 跑
```

Step 1-3 是**前置验证**,只验定价引擎可不可信。**这步通不过,后面没意义**。

## 不在本实验范围

```
v2 二叉树定价                  把欧式 BS 升级成带强赎/下修的美式
跨期对冲                       买 CB 同时空正股, delta-neutral
高频信号                      日内调仓
机器学习辅助估值              用历史数据训练修正 BS 偏差
组合优化                      不只是等权, 用 Markowitz / Black-Litterman
```

这些都是值得做的,但**先跑通 v1 简单版,看市场有没有真定价偏差**。如果 v1 都赚不到钱,说明这条路也不行,不用上 v2。
