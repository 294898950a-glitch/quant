# 可转债强赎策略 — 自动化优化循环设计

当前状态：主扫描管线 (main.py) 37s 跑完 341 只，有 Telegram 推送。
缺少：回测框架、参数优化引擎、Notion 知识积累。

## 目标架构

```
┌──────────────────────────────────────────────────┐
│  Cron: 每 30min（蒙特雷时区 8:30-15:00 交易时段）   │
└──────────────────────┬───────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────┐
│  optimizer.py (优化调度器)                          │
│  1. 读取当前最优参数（从 Notion 或本地 config）      │
│  2. 生成下一组候选参数（随机搜索 / 网格 / 贝叶斯）    │
│  3. 调回测引擎跑历史数据                             │
│  4. 评估结果（夏普、胜率、最大回撤）                  │
│  5. 如果优于当前最优 → 更新 config + 入库 Notion    │
└──────────────────────────────────────────────────┘
```

## 需要依次建造：

### Phase 1: 回测框架 (backtest.py)
- 从 akshare 拉历史日线数据（过去 90/180 天）
- 模拟每天扫描 → 生成信号 → 模拟买入/卖出
- 输出：总收益率、年化夏普、胜率、最大回撤、交易次数

### Phase 2: 参数优化 (optimizer.py)
- 可优化参数：Logit 系数 5 个、阈值 3 个、动量窗口、成交异常比
- 搜索策略：先随机搜索 50 组，再局部精调
- 自动对比 baseline（当前系数）vs 新参数

### Phase 3: Notion 入库
- 每次优化结果形成一条 Notion 记录
- Schema: 参数快照、回测指标、与 baseline 对比

### Phase 4: Cron 调度
- 蒙特雷时区 (America/Monterrey, UTC-6)
- 交易时段 8:30-15:00（对应上海 22:30-05:00）
- 每 30 分钟一个 iteration

---

## 每次迭代输出到 Notion Wiki 的格式

**条目类型**: Raw Inbox → Wiki 永久笔记
**标题**: "可转债策略优化 #N — YYYY-MM-DD HH:MM (Monterrey)"
**正文**:
- 本次迭代类型（调参/加特征/改逻辑）
- 参数快照（5 系数 + 3 阈值）
- 回测结果（Benchmark vs New）
- 是否接受（Accepted/Rejected + 理由）
- 下一步计划
