# 评测框架 — 设计 RFC

**起草于 2026-05-10. 目标: 给所有未来策略提供一套统一的"值不值得上 / 真信号还是过拟合 / LLM 调参对不对"的判定方法.**

---

## 一句话

不是新策略, 是给**判定策略好坏**这件事建一套客观、可重复的尺子.

## 为什么现在做

```
实验 #1 (网格 6 标的): 全失败. 但失败的判定靠的是事后人工对账(全段重跑 vs 持有不动).
                      没有事前可定义的"上 / 不上"阈值.
实验 #2 (cb_arb):     初始参数 4 年 +41% 跑赢 CB 等权 +30 个点.
                      LLM 调过 240 轮反而砍到 +14% / +3.5 个点.
                      LLM 看错指标 → 朝错方向调 → 把 alpha 砍掉.
                      
共同根因: 我们一直在用"单池 sharpe / 单池总收益"反馈 LLM,
         但这两个指标在熊市段(任何策略都亏)永远给负反馈,
         逼 LLM 学到"减仓避亏"这条死路, 跟我们的初衷相反.
         
评测标准没建好, 自循环就在原地打转, 越调越烂.
```

---

## 第一部分:评测框架要回答的 3 个问题

```
Q1. 这个策略 / 这组参数, 值不值得上?    → 给"是 / 否"明确答案 + 理由
Q2. 这组参数是真信号还是过拟合?        → 鉴定可信度
Q3. LLM 这一轮调参方向对不对?         → 单轮反馈给 LLM 看
```

每个问题都对应**一个可计算的判定函数**, 不是模糊判断.

---

## 第二部分:Q1 — 值不值得上

### 比的对象:5 个基准

```
1. 现金            (年化 1.5%, 无风险)
2. 持有 CB 等权     (公认 CB 基准, 4 年 +10.56%, 回撤 -29%)
3. 持有沪深 300    (大盘股票基准)
4. 持有红利组合     (神华+长电+工行+建行 等权, 实证 4 年 +100% 以上)
5. 60-40 配置      (60% 沪深 300 + 40% 国债)
```

每个基准都用**同一段时间 + 同一笔资金**模拟. 跟策略并排画累计净值 + 表格对比.

### 4 个判定维度

```
A. 总收益超额    策略累计收益 - 各基准累计收益
B. 风险超额     策略最大回撤 / 基准最大回撤
C. 跨年一致性   策略每个完整年份是否跑赢基准
D. 稳定性      不同分段 (按年 / 按市场状态) 表现的离散度
```

### 阈值 (上线标准)

```
要上一个策略, 必须同时满足:
  
  ✓ 跑赢"现金"和"CB 等权"两条线 (这是底线, 否则不如不做)
  ✓ 最大回撤 ≤ "CB 等权"基准的 1.2 倍 (风险约束)
  ✓ ≥ 60% 完整年份跑赢"CB 等权" (一致性约束, 防一年定胜负)
  
要上"值得真上"档, 还要:
  
  ✓ 跑赢"红利组合" (证明策略有真 alpha, 不是搭市场便车)
  
要上"明确好"档:
  
  ✓ 信息比率 ≥ 0.5 (年化超额 / 跟踪误差)
  ✓ 任何完整年份跌幅 ≤ 同年 CB 等权
```

### 输出形式

```
策略评测报告 (评测器自动生成, ~2 页):

1. 累计净值图 (策略 + 5 个基准并排画)
2. 关键指标表 (年化收益 / 最大回撤 / 月胜率 / 信息比率, 6 列)
3. 跨年一致性表 (每年策略 vs 5 个基准, 7 行 × 6 列)
4. 阈值判定 (打勾打叉, 给出"上 / 不上 / 还差什么")
5. 风险关注点 (回撤阶段定位, 哪些月份输得最多, 是不是同方向)
```

## 第三部分:Q2 — 真信号 vs 过拟合

### 切两段

```
训练段 (LLM 看过):       2022-01-01 ~ 2025-06-30 (3.5 年)
完全留出段 (LLM 永不见): 2025-07-01 ~ 2026-05-08 (10 个月)

注: 当前 cb_arb 的 8 池设计已用 2022-01 起做 OOS, 但所有 8 池 LLM 都见过.
   为做 Q2, 必须从最右端切出一段 LLM 永不见的部分, 上面只跑"挑出来的最优参数",
   不允许任何反向调.
```

### 鉴定规则

```
训练段挑出 "最优参数" (按 Q1 的阈值组合定义最优, 不是单一指标)
直接拿到留出段跑, 不再调

留出段总收益 ≥ 训练段总收益 × 80%      → 真信号
留出段总收益 ∈ [训练段 × 50%, × 80%]  → 弱信号 (谨慎上, 留口子)
留出段总收益 < 训练段 × 50%            → 过拟合 (不上)
留出段亏 / 训练段赚                    → 反转 (重写策略)
```

### 数据

```
当前 cb_warehouse 含 1012 CB, 数据覆盖 2017-? ~ 2026-05.
留出段 10 个月 (2025-07 ~ 2026-05) 数据已有, 可直接切.
```

### 接入点

```
1. 评测器读取 holdout.json 的 OOS 区间, 但内部再细切出"右尾留出段"
2. LLM 每轮拿不到的 trajectory + 最终选定参数后, 自动跑右尾留出段
3. 输出"留出段成绩 vs 训练段成绩"对比, 直接喂 Q1 阈值判定
```

## 第四部分:Q3 — LLM 单轮调参方向对不对

### 当前问题

```
LLM 看的是 oos_metrics["excess_return"] (单池 130 天的超额)
 → 在 pool 0 (2022 H1 熊市) 永远负 → LLM 学"减仓减仓"
 → 实际熊市段没人不亏, 这是市场状态决定的, 不是参数错
 → LLM 误判
```

### 改成

```
LLM 看的是 cumulative_metrics["excess_return"]:
  从第 1 池开始, 把所有跑过的池子 daily_returns 串起来, 跟同期 CB 等权 daily_returns 串起来
  累计差就是"我连续跑到现在 vs 持有 CB 不动"
  
  pool 0 跑完, LLM 看到: 累计超额 -2.5%
  pool 1 跑完, LLM 看到: 累计超额 -1.0% (恢复, 但还在水下)
  ...
  pool 6 跑完, LLM 看到: 累计超额 +18% (此时已大幅领先)
  
  这个累计指标对"市场状态"是单向稳定的:
  在熊市段累计超额平稳或缓降, LLM 不会过激;
  在牛市段累计超额上行, LLM 看到自己的方向是对的.
```

### 单轮反馈信号

```
改前累计 - 改后累计 = 这一轮调参的真实效果
  
> 0 (改后累计变高)  → 方向对, 鼓励同方向继续
< 0 (改后累计变低)  → 方向错, 提示反向 / 换参数维度
≈ 0                → 无效改动 (像 min_remaining_size 这种 1 维死锁的典型表现)
```

### 顺手治"无效改动"问题

```
当 LLM 连续 K 轮 (K=3) 调同一参数累计变化都 ≈ 0,
评测器自动给 hypothesizer 反馈:
"参数 X 在当前数据 / 阈值上不 binding, 请换维度"

具体到 cb_arb:
  min_remaining_size 在 [50M, 500M] 范围零过滤效果
  → LLM 改它累计无变化 → 评测器自动标 no-op → hypothesizer 跳过
  
等价于自动暴露了 01:15 CST REVIEW 提到的"Rule 2 一维死锁"问题, 不需要专门修 hypothesizer
```

## 第五部分:cb_arb 现有数据的回判

### 不重跑, 用现有数据重算

```
评测器实施第一阶段, 直接读:
  data/cb_arb/runs.jsonl (240 轮 + 后来 30 轮的所有 backtest 输出)
  data/cb_arb/sealed_pools.json (8 池区间)
  data/cb_warehouse/cb_basic.parquet (CB 主数据)

跑 3 件事:
  1. 按 Q1 重新评测 cb_arb 初始参数, 给"上 / 不上"
  2. 按 Q1 重新评测 cb_arb 240 轮末段参数, 给"上 / 不上"
  3. 按 Q3 重算 LLM 240 轮的累计超额轨迹, 看在每一轮 LLM 应该看到什么

预期结果 (假设评测框架算得对):
  初始参数: 跑赢现金 ✓, 跑赢 CB 等权 ✓, 但跑不赢红利组合 (年化 9% vs 红利 ~25%)
           判定: "底线档" — 可以上但不优秀
  
  240 轮末段: 跑赢现金 ✓, 但跑不赢 CB 等权 (年化 3.3% vs CB 等权 2.5%, 接近)
              判定: "弱信号" 或 "不上"
  
  Q3 累计超额轨迹: LLM 在 pool 0-3 (前半 OOS) 看到累计超额持续下行 →
                   实际数据上仍会建议减仓; 但 pool 4+ 累计反弹, LLM 应在该阶段
                   重新偏向激进. 累计指标的"逆周期性"会比单池稳, 但不能完全解决熊市段问题.
```

### 这步的作用

```
不是重跑 cb_arb, 是用旧数据验证评测框架本身. 验证 4 件事:
  - 评测器算的累计净值跟手算对得上
  - 阈值判定能给出明确"上 / 不上"
  - 留出段 (右尾 10 个月) 数据切得出来
  - LLM 累计超额轨迹算得出来 + 跟单池版本明显不同
```

## 第六部分:实施计划

### Step 1 — 评测器骨架 (~1.5 天)

```
新增模块: framework/evaluator.py (或 strategies/cb_redemption/evaluator.py 复用旧位置)

核心 API:
  EvaluationResult = evaluate(
      strategy_returns: pd.Series,  # 每日策略收益率
      benchmarks: dict[str, pd.Series],  # 基准收益率
      config: EvaluationConfig,
  )
  
  EvaluationResult 含:
    - cumulative_curves: 净值曲线 (策略 + 基准)
    - metrics_table: 6 列指标表
    - yearly_consistency: 跨年表
    - thresholds: 4 个阈值的判定结果
    - tier: "明确好" / "值得真上" / "底线档" / "不上"
    - reasons: 文字描述

新增模块: framework/benchmarks.py
  接口:  load_benchmark(name, start, end) -> pd.Series (daily_returns)
  实现:  
    - cash:         恒定 1.5%/250 daily
    - cb_equal:     当前已有的 _get_cb_index() 升级
    - csi300:       新接 akshare ak.index_zh_a_hist("000300")
    - dividend:     神华+长电+工行+建行 等权, akshare ak.stock_zh_a_hist 各拉一份
    - sixty_forty:  60% csi300 + 40% 国债 ETF (511010)

测试 (~10 个):
  - 5 个基准都能加载, 没 NaN, 长度对得上
  - cumulative_curves 起点都从 1.0 出发
  - tier 判定在边界值上正确
  - "明确好" 阈值的策略能识别
```

### Step 2 — 留出段切割 + Q3 反馈接入 (~1 天)

```
新增: framework/holdout_splitter.py
  接口:  split_holdout(start, end, holdout_months=10) -> (train_range, holdout_range)
  
新增到 verifier.py: cumulative_metrics 字段
  含: cumulative_total_return, cumulative_excess_return (vs cb_equal)
  按 sealed_pools.json 顺序, 把所有已跑过的池子 daily_returns 串接起来计算

修改 hypothesizer.py:
  prompt 显示的指标改用 cumulative_metrics 替代 oos_metrics 主位
  保留单池指标作为辅助, 但明确告知 LLM 主看哪个

修改 auditor.py:
  挖数据 / 没进展 的判定阈值用累计版本

测试 (~8 个):
  - cumulative_excess_return 单调可加 (每池增量等于该池超额)
  - holdout 切割正确, 不重叠
  - hypothesizer 收到的指标格式对
```

### Step 3 — 回判 cb_arb + 出报告 (~0.5 天)

```
新增脚本: scripts/evaluate_cb_arb_legacy.py
  读 240 轮 + 30 轮 runs.jsonl, 算两组参数的 Q1 评测
  读 8 池 OOS, 算 LLM 的累计超额轨迹 (Q3)
  
输出: reports/cb_arb_evaluation_2026-05-10.md
  含:
    - 初始参数 vs 5 基准对比表
    - 240 轮末段参数 vs 5 基准对比表
    - LLM 累计超额轨迹图 (240 轮)
    - 判定: cb_arb 是否值得上
```
