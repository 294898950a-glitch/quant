## 已确认的问题

1. 这 8 个池子的池内最好结果不能当最终结论。

   原因是每个池子的 best 都是在本池调出来的，可能只是适配本池。

2. 必须做 8 x 8 交叉评测。

   也就是 8 组 best 参数分别跑 8 个池子，得到 64 个结果。最终要看跨池平均、最差池、正超额数量，而不是只看本池最好。

3. 当前退出逻辑缺少“赚多少退出”和“盈利不足时不要因排名变贵过早卖”的控制。

   当前已有退出条件:

   - 强赎卖出
   - 最大持有天数卖出
   - 止损卖出
   - 排名变贵卖出
   - 回测结束平仓

   当前缺少:

   - `take_profit_pct`
   - `min_profit_for_rank_sell_pct`

4. 30 轮只适合第一阶段方向筛查。

   当前参数较多，30 轮不足以做最终调参结论。下一阶段如果新增退出参数，应提高到每池 80 到 120 轮。

## 结论

1. 没有一组参数能通吃 8 个池子。

2. pool_6 参数组按平均超额最好，但不是最稳的最终答案。它在 pool_6 表现很强，在 pool_0、pool_2、pool_4 为负。

3. pool_3 参数组在 pool_0 到 pool_4 上最强或接近最强，但在 pool_5 到 pool_7 上明显失效。这说明前后行情结构不同。

4. pool_5 到 pool_7 是强行情段，后半段参数组收益很高，但拿到前半段容易亏。不能直接用后半段高收益参数作为最终策略。

5. 下一阶段不应该只选单一 best 参数继续跑。应该增加路线判断:

   - 前半段行情适合 pool_3 风格参数。
   - 后半段行情适合 pool_6 / pool_7 风格参数。
   - 需要找一个能判断行情阶段的切换条件，或者构造更稳的折中参数组。

6. 退出逻辑仍是核心待修问题。当前交叉评测不能证明止盈缺口一定是唯一原因，但结果支持这个怀疑: 盈利段能跑出很高收益，弱行情段却容易提前或错误退出，说明退出规则需要单独建模。

前 5 个池子是弱市或熊市，后 3 个池子是强市。

### 2. 参数风格差异

弱市最强参数来自 pool_3:

- `rating_floor_int = 4`
- `max_holding_days = 60`
- `rank_sell_pct = 0.45`
- `vol_window_days = 30`
- `max_position_pct = 0.03`

这是一组明显偏防守的参数: 更高评级、更短持有、更少仓位。

强市较强参数来自 pool_6 / pool_7:

- pool_6: `rank_buy_pct = 0.25`, `max_position_pct = 0.06`, `stop_loss_pct = -0.10`
- pool_7: `vol_window_days = 70`, `max_position_pct = 0.06`

这是一组明显偏进攻的参数: 买得更宽或持仓更大，允许波动更大。

### 3. 退出原因观察

抽查 pool_3 防守参数:

- 在弱市池 0-4 上，主要退出原因是 `max_holding_days` 和 `stop_loss`。
- `rank_sell` 次数很少。
- 在强市池 5-7 上，pool_3 参数明显跑输。

抽查 pool_5 / pool_6 / pool_7 进攻参数:

- 强市中盈利主要来自 `max_holding_days` 和 `end_of_period`。
- `rank_sell` 触发次数很少，且触发时通常不是微利退出。
- 这说明“没吃到肉就 rank_sell 跑路”不是当前最主要证据。

### 4. 修正判断

原先怀疑“缺少止盈导致没吃到肉”仍然值得测试，但目前更强的证据是:

1. 策略缺少行情阶段判断。
2. 弱市需要防守参数。
3. 强市需要进攻参数。
4. 用单一参数跨所有阶段会牺牲一边。

下一阶段优先级应为:

1. 增加行情阶段判断。
2. 在不同阶段切换防守 / 进攻参数。
3. 再单独测试 `take_profit_pct` 和 `min_profit_for_rank_sell_pct` 是否能改善退出。

2. 弱市行为用 `pool_0` 能提升整体收益，但 64 组显示真正弱市最稳的是 `pool_3`。

待验证假设:

1. 市场状态不应只有弱 / 中 / 强三档，还需要单独识别“横盘偏弱 / 假修复”。

## 低估金额修复时间

修正口径:

之前用排名判断低估/高估是不对的。这里改成金额口径。

低估金额:

`(理论价 - 实际价) * 可买数量`

其中:

- 可投入资金 = 当前市场档位下的单只仓位资金。
- 可买数量 = `可投入资金 / 实际价 / (1 + 交易费)` 后向下取整。
- 低估金额 > 0，表示理论价高于市场价。
- 低估金额 <= 0，表示已经修复到合理或高估。

统计对象:

- 区间: 2019-2024。
- 只看当前系统能进入估值计算的债。
- 记录每次从低估金额 > 0 开始，到低估金额 <= 0 的时间。

总体结果:

- 完成修复样本数: 3743。
- 修复到合理/高估的交易日数:

| 分位 | 天数 |
| --- | ---: |
| 25% | 1 |
| 50% | 4 |
| 75% | 23 |
| 90% | 87 |
| 95% | 156 |

按初始低估金额占可投入资金比例分组:

| 初始低估比例 | 样本数 | 中位天数 | 75%天数 | 90%天数 |
| --- | ---: | ---: | ---: | ---: |
| <=1% | 2359 | 2 | 8 | 33.2 |
| 1%-3% | 813 | 7 | 28 | 79.6 |
| 3%-5% | 184 | 25 | 70 | 122.4 |
| 5%-10% | 154 | 39.5 | 84 | 146 |
| 10%-20% | 116 | 103.5 | 163 | 243 |
| >20% | 117 | 201 | 284 | 325.8 |

结论:

1. 小低估修复很快，很多只是几天内消失。
2. 大低估修复明显更慢。
3. 如果初始低估金额超过可投入资金的 10%，90 天持有期通常不够。
4. 因此持有期不应该是固定 90 天，而应和初始低估金额挂钩。
5. 这不是追历史赢家，而是按价值回归所需时间设置等待成本。

## 止损复查规则试跑

理论判断:

触发止损后，已经亏掉的部分是沉没成本。此时不应该因为亏损本身卖出，而应该重新比较:

1. 当前持仓从现价到理论价的剩余低估金额。
2. 新候选从现价到理论价的低估金额。
3. 新候选是否超过当前持仓加换仓成本。

试跑规则:

- 价格触发止损时，不直接卖。
- 如果当前持仓低估金额 <= 0，卖出。
- 如果当前持仓低估金额仍 > 0，进入止损复查。
- 只有当新候选低估金额 > 当前持仓低估金额 + 3%仓位资金，才卖出换仓。

## 止损复查加入低估剩余比例

新判断:

触发止损时，如果低估金额从买入时明显收缩，说明买入理由已经削弱。不能只因为剩余低估金额仍为正就继续拿。

测试规则:

- 触发止损时，计算:

`止损日低估金额 / 买入日低估金额`

- 如果比例低于门槛，卖出。

## 止损触发后的重新估值测试

修正判断:

更合理的统一规则:

1. 买入时使用正常估值。
2. 触发止损时，说明市场已经强烈反对原判断。
3. 此时用更保守的估值参数重新计算低估金额。
4. 保守估值下仍有足够低估，才继续持有或只在更好机会出现时换仓。
5. 保守估值下低估消失或明显削弱，则卖出。

- 买入仍按正常绝对低估金额排序。
- 触发止损后，要求保守估值下低估金额至少保留买入低估金额的 30%。

- 不是所有恐慌日都应该降低期权价值。
- 不是所有期权型持仓都应该降低期权价值。
- 只有当持仓触发重新判断时，才应该用恐慌重估值参与机会成本比较。
- 恐慌重估值不能只由债底距离决定，还需要加入正股支持和替代机会。

下一步约束:

新策略应改成“触发式重估”:

- 正常日不折扣。

## 恐慌识别器第一版

目的:

把“真正恐慌触发”和“弱市状态”拆开看，避免后续策略把普通弱市也当成恐慌处理。

识别器输入:

- 转债等权价格变化。
- 当天上涨比例。
- 20 日上涨比例。
- 60/120 日回撤。
- 成交额相对过去 252 日的位置。

第一版分类:

- `shock_day`: 当天转债等权价格大跌，或当天跌幅较大且短期/宽度明显恶化。

- `panic_v1`: `shock_day` 或 `cb_liquidity_panic`。

### 强信号结论复验

复验目标:

- 验证“强信号机会名单更稳”是否只是汇总指标巧合。
- 验证普通信号机会名单为什么会伤害验收段。
- 验证强信号修 2020 的力度为什么不如普通信号。

复验结果:

| 对比 | 结论 |
| --- | --- |
| 强信号机会名单 vs 当前最好版本，验收段 | 资金曲线完全一致，差额天数 0，最终差额 0 |
| 普通信号机会名单 vs 当前最好版本，验收段 | 从 2025-09-10 开始改变路径，最终少 26935.09 |
| 普通信号机会名单 vs 当前最好版本，2020 | 最终多 36841.13 |
| 强信号机会名单 vs 当前最好版本，2020 | 最终多 19737.09 |

验收段细节:

- 强信号机会名单没有真正改变验收段收益路径，只改变了一笔退出标签。
- 2025-04-08 `113667.SH 春23转债`，当前最好版本记为 `stop_loss_value_gone`，强信号版本记为 `stop_loss_switch_value_gap`，亏损金额同为 `-3687.94`。
- 因此“强信号没有损害验收段”成立，而且不是汇总误差。

普通信号机会名单的问题:

- 普通信号触发太宽，会在验收段正常波动中改变持仓。
- 验收段从 2025-09-10 起资金曲线开始偏离，最终比当前最好版本少 `26935.09`。
- 主要表现为错过后续盈利交易，例如 `123247.SZ 万凯转债`、`123241.SZ 欧通转债`、`127084.SZ 柳工转2`，同时增加若干止损换仓亏损。

2020 细节:

- 普通信号机会名单修 2020 更强，因为它覆盖了更多普通恐慌日。
- 强信号机会名单修 2020 较弱，因为它漏掉一部分 2020 有保护价值的普通恐慌日。
- 2020 普通信号独有日期包括 `20200309`、`20200413`、`20200907`、`20200924` 等；这些日期可能是下一步寻找中间触发标准的重点。

复验后的判断:

1. 原结论基本正确: 强信号机会名单是更稳的默认候选，因为它不损害验收段。
2. 需要修正一点: 强信号并没有提升验收段，只是保持不变。
3. 普通信号能更好修 2020，但会伤害验收段，不能直接作为默认方案。
4. 下一步不应继续扩大保护，而应寻找“中间触发标准”: 多抓住 2020 有用日期，同时避开 2025-09-10 之后的误触发。

### 中等恐慌信号搜索

目标:

- 固定现有保护规则，只搜索“什么时候启动保护”。
- 目标不是扩大保护，而是补上 2020 有用日期，同时避开 2025-09 之后的误伤日期。

固定规则:

- 恐慌日仍使用训练好的恐慌识别器做期权重估。
- 机会名单只保护期权低估标的。
- 保护期 20 天。
- 连续 5 天低估不成立则移出保护。
- 连续 2 天非机会日则退出保护。
- 换仓要求新标的比当前标的明显更好，门槛为 12%。

搜索方式:

- 先按触发日期筛选 96 种不同日期组合。
- 再选 24 种做完整回测。
- 比较当前最好版本、普通机会名单、强机会名单、中等机会名单。

结果:

| 版本 | 训练超额 | 验收超额 | 2020 超额 | 2020 触发日 | 2025 触发日 |
| --- | ---: | ---: | ---: | ---: | ---: |
| 当前最好版本 | 0.094262 | 0.492618 | -0.178799 | - | - |
| 普通信号机会名单 | 0.069963 | 0.465683 | -0.141958 | 26 | 17 |
| 强信号机会名单 | 0.076116 | 0.492618 | -0.159062 | 14 | 4 |
| 中等信号机会名单 | 0.167085 | 0.492618 | -0.138588 | 19 | 5 |

中等信号规则:

- 当日跌幅小于等于 `-3%`。
- 或当日跌幅小于等于 `-1.6%`，且当日上涨比例小于等于 `20%`。
- 或 5 日跌幅小于等于 `-4%`，20 日跌幅小于等于 `-4%`，且 20 日上涨比例小于等于 `30%`。
- 信号次日生效。

中等信号日期:

- 2020: `20200203`, `20200204`, `20200302`, `20200317`, `20200324`, `20200407`, `20200413`, `20200427`, `20200508`, `20200716`, `20200717`, `20200727`, `20200910`, `20200911`, `20200914`, `20200925`, `20201026`, `20201102`, `20201214`
- 2025: `20250303`, `20250408`, `20250409`, `20250828`, `20250903`

判断:

1. 中等信号是当前最优方向: 验收段不下降，2020 修复超过普通信号，训练段也更好。
2. 关键不是大幅扩大保护，而是把当日下跌触发从 `-1.8%` 放宽到 `-1.6%`，同时仍要求上涨比例不高于 `20%`。
3. 20 日上涨比例从 `25%` 放宽到 `30%` 有帮助，但不能继续无约束放宽。
4. 这个结果比普通信号更合理，因为它没有命中之前造成验收伤害的 2025-09 之后误伤日期。
