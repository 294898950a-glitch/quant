# Quant Wiki Schema
# =============================================
# 本文件是量化知识库的"宪法"
# 定义了知识库的结构、约定和工作流
# LLM 在执行所有操作时必须遵循此文件
# =============================================

## 项目概述

基于 Karpathy llm-wiki 模式构建的**量化投资研究知识库**。
目标：将散落的量化资料（课程、论文、策略、回测、复盘）系统化为一个**持续增长、可交叉引用的知识资产**。

## 三层架构

### Layer 1: Raw Sources (原始资料) — `data/raw/`
不可修改的真相来源。LLM 只读不写。

目录规范：
```
data/raw/
├── joinquant/          # 聚宽平台资料
│   ├── courses/        # 课程内容
│   ├── api-docs/       # API 文档
│   └── strategies/     # 策略示例
├── papers/             # 学术论文
├── books/              # 书籍（如《人月神话》等）
├── web-articles/       # 网页文章存档
├── backtests/          # 回测结果数据
└── market-data/        # 市场数据
```

命名规范：`{source}_{topic}_{date}.{ext}`
例：`joinquant_均线策略入门_20260421.md`

### Layer 2: Wiki (知识层) — Notion
LLM 全权维护的结构化知识库。所有页面由 LLM 创建和更新。

#### Notion 页面层级
```
投资思考 (根页面)
├── quant-wiki-index          # 知识库总索引 (index)
├── quant-wiki-log            # 操作日志 (log)
├── entities/                 # 实体页面
│   ├── 指标/                  # MACD, RSI, 布林带, ...
│   ├── 策略/                  # 双均线, 海龟, 动量, ...
│   ├── 概念/                  # Alpha, Beta, 夏普比率, 最大回撤, ...
│   └── 市场/                  # A股, 美股, 期货, ...
├── sources/                  # 来源摘要页
│   └── 每个原始资料一篇摘要
├── syntheses/                # 综合分析页
│   ├── 策略对比/
│   ├── 因子研究/
│   └── 复盘总结/
└── tools/                    # 工具页
    ├── 监控面板/
    └── 计算器/
```

#### 页面模板类型

| 类型 | 用途 | Icon | 必需字段 |
|------|------|------|----------|
| Entity | 实体（指标/策略/概念） | 🏷️ | 名称、定义、公式、相关链接 |
| Source | 来源资料摘要 | 📄 | 标题、作者、日期、URL、关键要点 |
| Synthesis | 综合分析 | 🔬 | 标题、涉及实体、结论、数据支撑 |
| Log | 操作记录 | 📝 | 时间戳、操作类型、描述 |

### Layer 3: Schema (本文件)
定义以上两层的结构和规则。

## 操作流程

### Ingest (摄入)

当有新的原始资料需要入库时：

1. **放入 Raw**: 将资料放入 `data/raw/` 对应子目录
2. **阅读理解**: LLM 阅读全文，提取关键信息
3. **创建 Source 页面**: 在 Notion `sources/` 下创建摘要页
4. **更新/创建 Entity 页面**: 提取中的实体（指标、概念），更新对应 Entity 页
5. **更新 Index**: 更新总索引
6. **记录 Log**: 追加操作日志

输入：
```
ingest <file_path> [--batch] [--supervised]
```

输出：
- 1 个 Source 页面（资料摘要）
- N 个 Entity 页面（新建或更新）
- Index 更新
- Log 条目

### Query (查询)

用户提问时：

1. **查 Index**: 先读索引找到相关页面
2. **深入阅读**: 打开相关页面获取详情
3. **综合回答**: 交叉引用多个页面生成答案
4. **可选: 存回**: 如果回答本身有价值，存为新页面

### Lint (健康检查)

定期运行（建议每周一次）：

- [ ] 矛盾检测：不同页面是否有冲突说法
- [ ] 过时标记：是否有被新资料推翻的旧结论
- [ ] 孤儿页面：没有入链的页面
- [ ] 缺失实体：提到但没有独立页面的概念
- [ ] 缺失交叉引用：应该关联但未关联的页面
- [ ] 数据缺口：可以通过搜索补充的内容

## 索引与日志

### Index (quant-wiki-index)
Notion 页面，内容导向的目录。
格式：每个条目 = 链接 + 一行摘要 + 元数据（标签、日期）
按分类组织：Entities / Sources / Syntheses

### Log (quant-wiki-log)
Notion 页面，时间线式的操作记录。
格式：`## [YYYY-MM-DD HH:MM] {operation} | {description}`

可解析前缀：
- `[ingest]` — 新资料摄入
- `[query]` — 用户查询
- `[lint]` — 健康检查
- `[update]` — 页面更新
- `[synthesis]` — 综合分析

## 内容质量标准

### Source 页面必须包含
- ✅ 原始资料的完整引用（标题、作者、来源 URL、日期）
- ✅ 3-7 个关键要点（bullet points）
- ✅ 对现有 wiki 的增量更新说明（这个资料新增/改变了什么）
- ✅ 与相关 Entity 的链接

### Entity 页面必须包含
- ✅ 清晰的定义（一句话能说清楚）
- ✅ 公式/计算方法（如适用）
- ✅ 适用场景和局限
- ✅ 相关策略和指标的双向链接
- ✅ 来源引用（哪些资料讨论了这个实体）

### 禁止事项
- ❌ 不加来源地断言事实
- ❌ 创建孤立的页面（无链接）
- ❌ 删除已有内容只追加新视角（用删除线标注过时信息）
- ❌ 在 Entity 页面大段复制原文（提炼后引用）
