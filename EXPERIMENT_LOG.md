# 自循环量化框架 — 第一次实验封档

**时段**: 2026-04-21 ~ 2026-05-09
**结论**: 框架建成,网格策略在测试的所有标的上失败

---

## 框架建了什么

9 个角色的自循环系统(策略给定参数 → 自己跑回测 → 看结果 → 出主意 → 改参数 → 再跑,默认无人值守):

```
1. 测试器                跑回测出客观分数
2. 体检员                每轮硬规则 + 事件触发 LLM 评估参数合理性
3. 诊断员                看一次结果只描述事实
4. 审计员                看 N 轮趋势, 抓挖数据/没进展, 有否决权
5. 出主意者 (DeepSeek)    LLM 出具体建议, 中文 reason, 死格式
6. 编辑器                唯一允许写参数的入口, 范围+理由硬卡
7. 记忆员                每轮存档 + 防 AI 重复试同一方向
8. 总指挥                串起来的状态机 + 自动恢复
9. 市场画像员             给 LLM 看的原始统计 (无标签)
```

部署在 tencent-sig-vm 上,systemd-user 服务跑,outbox 接 telegram 推送(filter 默认静默,只在 6 类事件触发时推 + 全中文无术语)。

代码全部在 `strategies/cb_redemption/`(历史命名,实际是 framework 模块)。

---

## 测试的策略 + 结果

**统一策略**: 网格交易(grid)
**统一回测**: 8 个隔离样本袋,OOS = 2022-01-01 ~ 2026-05-08(每袋 ~130 天,各跑 30 轮调参)
**评估指标**: 全段重跑 OOS 后的总收益 + 跟"直接持有不动"比

| 策略         | 标的            | 平均得分 | 历史最高 | 4 年全段重跑 vs 持有不动 |
|-------------|----------------|---------|---------|------------------------|
| sp500-grid  | 513500.SH 标普500 ETF       | +0.22 | +4.84 | 平均不行,偶尔好运 |
| csi500-grid | 510500.SH 中证500 ETF       | +1.90 | +4.18 | **审计员判过拟合,不可信** |
| yzm-grid    | 300415.SZ 伊之密(单股)      | +0.42 | +2.17 | 单股波动太大被打穿 |
| shenhua-grid| 601088.SH 中国神华          | +1.31 | +3.34 | grid +96.67% **vs 持有 +167.02% (输 70 点)** |
| yangtze-grid| 600900.SH 长江电力          | +0.50 | +2.83 | (未跑全段对比) |
| icbc-grid   | 601398.SH 工商银行          | +0.78 | +2.52 | grid +0.15% **vs 持有 +102.72% + 股息 (输 102 点)** |

## 结论

### 1. 网格交易方法论在 2022-2026 的中美股票市场不成立

```
shenhua: 全段输持有 70 点
icbc:    全段输持有 102 点 (LLM 调到几乎不持仓, 错过整段牛市)
其余:    OOS 评分都不达"应该上"标准
```

不是参数问题,是**策略 × 市场环境**根本错配:这 4 年里 6 个标的里至少 5 个(神华、工行、长电、yzm、sp500、csi500)都在涨,网格抓震荡的逻辑没生存空间。

### 2. "高分"陷阱:某 130 天窗口的最高分都是过拟合

工行 iter 111 跑出 +2.52 的"高分"用全段重跑:**+0.15% 全程**。

LLM 在 240 轮调参里找到的不是"赚钱方法",是"恰好在那 130 天窗口表现好的参数"。这是过拟合的教科书表现 — 也正因此设计了**隔离样本袋 + 审计员**,csi500 的 +1.90 平均分被审计员正确判定为挖数据,事实证明该判得对。

### 3. 框架本身有效

```
✓ 1440 次回测 (240 轮 × 6 策略) 跑完无崩溃
✓ 审计员真的抓到 csi500 过拟合
✓ 体检员拦下过参数 fatal 组合
✓ LLM 中文 reason 让你看清"为什么不行"
✓ 6 个不同标的的横向对比, 一致结论 → 不是运气, 是结构
```

## 如果再开

不要再做网格。值得试的方向(按 grid 失败的反向):

```
趋势跟随(动量)         跟着上涨买, 反向不买. 跟"网格在涨势卖光"反着
事件驱动                 抓并购/分红/重组的信息差, 不抓震荡
量价背离                 抓"主力出货"信号
跨品种套利               不依赖单标的方向
```

不必立刻动。这次实验的最大价值是**框架已经成熟,下次想试新策略复用就行**。
